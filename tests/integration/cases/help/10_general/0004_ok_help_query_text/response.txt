Command
  query: read many entities with structural filters and deterministic pagination

Syntax
  spec-cli query [options]

Schema
  ResolvedPath: /spec-cli-fixed-root/spec.schema.yaml
  Status: loaded
  CLI-oriented projection:
    version: "0.0.4"
    description: Workspace specification schema
    entity:
      feature:
        description: Feature specifications
        idPrefix: FEAT
        meta:
          status:
            description: Lifecycle status
            required: false
            schema:
              type: string
              enum:
                - draft
                - active
                - deprecated
          watchers:
            description: Related services
            required: false
            schema:
              type: array
              items:
                type: entityRef
                refTypes:
                  - service
        refs:
          owner:
            description: Parent service
            required: ${meta.status == 'active'}
            schema:
              type: entityRef
              refTypes:
                - service
        content:
          sections:
            summary:
              description: Short summary
              required: ${meta.status == 'active'}
              title:
                - Summary
      service:
        idPrefix: SVC
        meta:
          tier:
            required: true
            schema:
              type: string

Options
  --type <entity_type>
    takes_value: true
    required: false
    repeatable: true
    mutually_exclusive_groups: none
    schema_derived: true
    schema_derivation: entity type keys from Schema.entity
    description: Early entity type filter.
  --where <expr>
    takes_value: true
    required: false
    repeatable: false
    mutually_exclusive_groups: none
    schema_derived: true
    schema_derivation: JMESPath expression over query item schema
    description: Schema-aware JMESPath filter expression.
  --select <field>
    takes_value: true
    required: false
    repeatable: true
    mutually_exclusive_groups: none
    schema_derived: true
    schema_derivation: selectors from CLI projection-namespace
    description: Projection-namespace selector.
  --sort <field[:asc|desc]>
    takes_value: true
    required: false
    repeatable: true
    mutually_exclusive_groups: none
    schema_derived: true
    schema_derivation: sort fields from CLI filter-namespace
    description: Deterministic sort term.
  --limit <n>
    takes_value: true
    required: false
    repeatable: false
    mutually_exclusive_groups: none
    schema_derived: false
    schema_derivation: none
    description: Page size; integer >= 0; default 100.
  --offset <n>
    takes_value: true
    required: false
    repeatable: false
    mutually_exclusive_groups: none
    schema_derived: false
    schema_derivation: none
    description: Page offset; integer >= 0; default 0.

Rules
  - projection-namespace argument is --select.
  - filter-namespace arguments are --sort and --where.
  - projection-namespace selectors: type, id, slug, revision, createdDate, updatedDate, meta.<name>, refs, refs.<field>, content.raw, content.sections, content.sections.<name>.
  - filter-namespace sort paths: type, id, slug, revision, createdDate, updatedDate, meta.<name>, refs.<field>.resolved, refs.<field>.type, refs.<field>.id, refs.<field>.slug, refs.<field>.reason, content.raw, content.sections.<name>.
  - --type uses schema-derived entity type keys and is not part of projection-namespace or filter-namespace.
  - If --select is omitted, default projection is type, id, slug, meta and refs.
  - Query guarantees deterministic ordering.
  - If --sort is omitted, effective sort is type:asc, then id:asc.
  - If --sort is provided, effective sort is user sort plus hidden tail type:asc, then id:asc, unless that tail is already present at the end.
  - --where uses JMESPath truthiness over schema-aware query-item context.
  - --where forbids content root access except content.sections... and forbids content.raw.
  - --where forbids meta.<field> access for entityRef fields; use refs.<field>.

Examples
  spec-cli query --type feature --select id --select meta.status
  spec-cli query --where "meta.status == 'active'"
  spec-cli query --where "refs.owner.id == 'SVC-2'"
  spec-cli query --where "contains(content.sections.summary, 'retry')"
  spec-cli query --where "length(refs.watchers[?reason == 'missing']) > `0`"
  spec-cli query --sort updatedDate:desc --limit 50 --offset 0
