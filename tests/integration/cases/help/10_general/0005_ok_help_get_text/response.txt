Command
  get: read one entity by exact id

Syntax
  spec-cli get [options] --id <entity_id>

Schema
  ResolvedPath: /spec-cli-fixed-root/spec.schema.yaml
  Status: loaded
  CLI-oriented projection:
    version: "0.0.4"
    description: Workspace specification schema
    entity:
      feature:
        description: Feature specifications
        id_prefix: FEAT
        meta:
          status:
            description: Lifecycle status
            required: false
            schema:
              type: string
              enum:
                - draft
                - active
                - deprecated
          watchers:
            description: Related services
            required: false
            schema:
              type: array
              items:
                type: entity_ref
                refTypes:
                  - service
        refs:
          owner:
            description: Parent service
            required:
              when:
                eq?: [meta.status, active]
            schema:
              type: entity_ref
              refTypes:
                - service
        content:
          sections:
            summary:
              description: Short summary
              required:
                when:
                  eq?: [meta.status, active]
              title:
                - Summary
      service:
        id_prefix: SVC
        meta:
          tier:
            required: true
            schema:
              type: string

Options
  --id <entity_id>
    takes_value: true
    required: true
    repeatable: false
    mutually_exclusive_groups: none
    schema_derived: false
    schema_derivation: none
    description: Exact target id.
  --select <field>
    takes_value: true
    required: false
    repeatable: true
    mutually_exclusive_groups: none
    schema_derived: true
    schema_derivation: read selectors from CLI read-namespace
    description: Read-namespace projection selector.

Rules
  - Read-namespace leaf paths: type, id, slug, revision, created_date, updated_date, meta.<name>, refs.<field>.type, refs.<field>.id, refs.<field>.slug, content.raw, content.sections.<name>.
  - --select uses read-namespace leaf paths.
  - --id addresses one entity by exact id and returns a single-entity read result.
  - If --select is omitted, default projection is type, id, slug, revision and meta.
  - If selected content.sections.<name> is absent, the field must be present with null value.
  - Existing but invalid target data must not degrade into false ENTITY_NOT_FOUND when deterministic read is still possible.

Examples
  spec-cli get --id FEAT-8
  spec-cli get --id FEAT-8 --select id --select meta.status
